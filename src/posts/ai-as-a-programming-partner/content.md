# AI as a Programming Partner

Using AI while learning to code is strange because it can make you feel powerful and suspicious at the same time.

On a good day, it feels like having a patient teammate beside you. You can ask the messy question. You can say, “I do not understand why this broke,” or “Can you explain this like I am building it for the first time?” You can get unstuck faster than you would alone.

But there is a trap in that speed.

If AI gives me the answer before I understand the problem, I can move forward without actually learning the path. The code may work, but my confidence does not grow because I did not earn the mental map.

So I am trying to use AI differently.

Instead of only asking for solutions, I ask for explanations. I ask it to compare two approaches. I ask what the code is doing line by line. I ask what could go wrong. I ask for the smallest possible version first.

The best use of AI, for me, is not “do this for me.”

It is “walk beside me while I learn how this works.”

That distinction matters.

Because the goal is not to produce code I cannot maintain. The goal is to become the kind of person who can read, question, revise, and eventually build with more independence.

AI can help with that.

But I still need to stay in the room.
